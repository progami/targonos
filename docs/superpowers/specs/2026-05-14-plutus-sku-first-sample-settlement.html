Target Flow

No brand-level QBO accounts. Plutus owns SKU and PO truth. QBO holds clean accounting output and trace references.

1. Amazon source SP-API settlement events, fee reports, AWD reports, ads evidence, reimbursements, reserves.
2. Plutus classify Classify settlement lines to accounting categories; resolve SKU/PO only for inventory units and COGS.
3. Inventory ledger Apply PO cost layers, receipts, sales, returns, removals, and FIFO consumption.
4. QBO posting Post to simple accounting accounts with PLUTUS_REF and PLUTUS_LINE trace fields.
5. Sellerboard COGS Export SKU-level COGS batches from Plutus cost layers and QBO source evidence.

Classification Rules

Settlement lines post by category. Only inventory-affecting rows need SKU/PO assignment; ambiguity blocks COGS, not the whole settlement category posting.

INVENTORY_SKU Inventory unit or cost row has SKU, ASIN, or known alias. Plutus maps it only for inventory COGS.
COST_ALLOCATED Inventory cost row has no SKU, but a deterministic QBO/PO source allocates it to SKUs. Allocation source is stored.
NON_INVENTORY Sales, fees, ads, tax, reserves, rollovers, and settlement-control rows post to simple category accounts with no Plutus SKU P&L.
TREATMENT_REVIEW Missing SKU, missing PO cost layer, negative inventory, missing allocation source, or QBO drift blocks COGS/Sellerboard COGS output.

## Repro Routes
- `/hermes/orders`: request review on a US marketplace order, requeue a request-review dispatch, or run backfill with review automation enabled.
- `/hermes/reviews`: export a manual-review dataset larger than 1000 rows.
- `/hermes/campaigns`: inspect campaign listings while multiple seller connections have campaigns.
- `/hermes/campaigns/new`: configure Test-tab values, save, then inspect whether any experiment was created.
- `/hermes/experiments`: inspect creation flow and experiment summary values.
- `/hermes/accounts`: compare account list status with explicit account test results.
- `/hermes/templates`: inspect whether templates are loaded from or persisted through any API-backed workflow.

## Confirmed Issues
- US request-review workflow is hard-disabled. [`lib/amazon/policy.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/lib/amazon/policy.ts) disables the US marketplace, [`api/solicitations/request-review/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/solicitations/request-review/route.ts) returns `409` when that rule fails, [`api/dispatches/requeue/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/dispatches/requeue/route.ts) blocks the same path, [`server/orders/ingest.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/orders/ingest.ts) silently skips US orders during automated enqueue, and [`server/dispatch/ledger.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/dispatch/ledger.ts) throws if a US review request is queued. Manual, requeue, and backfill-driven review workflows do not function for US accounts.
- Campaigns and experiments are not scoped to the active connection. [`campaigns/page.tsx`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/campaigns/page.tsx) fetches `/api/campaigns` without a connection filter, [`api/campaigns/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/campaigns/route.ts) returns `listCampaigns()` globally, and [`server/campaigns/crud.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/campaigns/crud.ts) selects all campaigns. The same global pattern exists for experiments in [`api/experiments/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/experiments/route.ts) and [`server/experiments/crud.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/experiments/crud.ts). A multi-connection Hermes workspace can display another seller’s campaign/test state.
- Campaign creation discards test configuration instead of persisting an experiment. [`campaigns/new/page.tsx`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/campaigns/new/page.tsx) collects holdout and test settings, but submit only posts campaign fields to [`api/campaigns/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/campaigns/route.ts) and redirects back to `/campaigns`. [`server/campaigns/crud.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/campaigns/crud.ts) only creates the campaign row. The Test tab has no persistence path.
- Experiment workflow lacks a usable creation path and core server validation. [`experiments/page.tsx`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/experiments/page.tsx) renders a `New test` button with no navigation or submit handler, while [`api/experiments/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/experiments/route.ts) accepts creation without verifying allocation totals or campaign existence and [`server/experiments/crud.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/experiments/crud.ts) stores raw allocations unchanged.
- Order backfill cannot attribute automated review dispatches to campaign, experiment, or template state. [`api/orders/backfill/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/orders/backfill/route.ts) accepts schedule controls but no `campaignId`, `experimentId`, or `templateId`. [`server/orders/ingest.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/orders/ingest.ts) can carry those fields into queued review dispatches, but the backfill route never passes them.
- Review export silently truncates large datasets. [`api/reviews/export/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/reviews/export/route.ts) requests up to `5000` rows, but [`server/reviews/query.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/reviews/query.ts) clamps the effective limit to `1000`. Large exports are incomplete with no corresponding export-route warning.
- Account list status does not reflect live connectivity. [`api/accounts/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/accounts/route.ts) marks an account `connected` if configuration loads, while the actual live SP-API check only happens in [`api/accounts/test/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/accounts/test/route.ts). Hermes can present an account as connected even when real API operations would fail.
- Templates are not backed by a workflow. [`templates/page.tsx`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/templates/page.tsx) uses hardcoded template data and does not fetch, create, update, or delete through any reviewed Hermes API route. The Templates surface is not managing persisted template state.

## Likely Root Causes
- Hermes still contains placeholder marketplace policy logic. [`lib/amazon/policy.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/lib/amazon/policy.ts) explicitly labels the marketplace switches as placeholders, and that placeholder now gates the full US review workflow.
- Campaign and experiment persistence was only partially implemented. The create/list APIs in [`api/campaigns/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/campaigns/route.ts) and [`api/experiments/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/experiments/route.ts) store broad JSON payloads but do not enforce workflow invariants or connection scoping.
- Hermes models adjacent workflow surfaces but does not carry shared identifiers across them. The clearest break is between [`api/orders/backfill/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/orders/backfill/route.ts) and [`server/orders/ingest.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/orders/ingest.ts), where dispatch attribution fields exist in the server path but are unreachable from the route.
- Account health is split between “configured” and “verified” logic, but the list endpoint in [`api/accounts/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/accounts/route.ts) presents the weaker signal as if it were operational health.

## Recommended Fixes
- Replace the placeholder US marketplace rule in [`lib/amazon/policy.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/lib/amazon/policy.ts) with verified product policy, then align manual request-review, requeue, and backfill behavior to that single rule.
- Scope campaign and experiment queries by active connection in both the route handlers and CRUD helpers.
- Complete the campaign creation workflow in [`campaigns/new/page.tsx`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/campaigns/new/page.tsx) so Test-tab state creates or updates experiment records instead of being discarded.
- Add server-side experiment invariants in [`api/experiments/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/experiments/route.ts): valid campaign reference, valid allocation totals, and required control/variant structure.
- Extend [`api/orders/backfill/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/orders/backfill/route.ts) to accept and pass campaign, experiment, and template identifiers into [`server/orders/ingest.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/server/orders/ingest.ts).
- Make review export honest end to end: either support the requested export size or surface the real cap explicitly in [`api/reviews/export/route.ts`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/api/reviews/export/route.ts).
- Unify account health semantics so the account list distinguishes “configured” from “verified” or performs the live test before returning `connected`.
- Either wire [`templates/page.tsx`](/Users/jarraramjad/dev/targonos-main/apps/hermes/src/app/templates/page.tsx) to persistent template APIs or remove it from active workflow coverage until that path exists.
